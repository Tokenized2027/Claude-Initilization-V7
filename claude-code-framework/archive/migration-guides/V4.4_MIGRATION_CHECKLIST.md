# v4.4 Migration Checklist

> **Use this checklist to track your v4.4 adoption progress**

---

## Pre-Migration

- [ ] Read `V4.4_RELEASE_SUMMARY.md` (15 min)
- [ ] Read `UPGRADING_V4.3_TO_V4.4.md` (15 min)
- [ ] Backup current project: `cp -r .claude .claude-backup-$(date +%Y%m%d)`
- [ ] Verify git status: `git status` (commit any uncommitted work)

**Estimated time:** 30 minutes

## Week 1: Memory System

### Day 1: Setup (30 minutes)

- [ ] Create memory directory: `mkdir -p .claude/memory/cross-project`
- [ ] Copy memory templates:
  ```bash
  cp ~/mastering-claude-code/toolkit/templates/memory/* .claude/memory/
  ```
- [ ] Or create manually if templates unavailable:
  - [ ] `.claude/memory/decisions.json`
  - [ ] `.claude/memory/patterns.md`
  - [ ] `.claude/memory/preferences.md`
  - [ ] `.claude/memory/project-context.json`
  - [ ] `.claude/memory/session-history.json`

### Day 1: Configure (15 minutes)

- [ ] Edit `.claude/memory/preferences.md`:
  - [ ] Add coding style preferences
  - [ ] Add framework/tool preferences
  - [ ] Add communication preferences
  - [ ] Add workflow preferences

### Day 1: Initialize (15 minutes)

- [ ] Start Claude Code
- [ ] Initialize memory with current project state
- [ ] Create first session-history entry
- [ ] Verify all memory files populated

### Days 2-7: Use Memory (daily)

- [ ] **Day 2:** Use memory for all sessions
- [ ] **Day 3:** Add first decision to memory
- [ ] **Day 4:** Add first pattern to memory
- [ ] **Day 5:** Review session history
- [ ] **Day 6:** Test cross-project sync (if multiple projects)
- [ ] **Day 7:** Verify 90% faster session starts

### Week 1: Validation

- [ ] Run: `./toolkit/validate-v4.4.sh`
- [ ] All memory files exist and populated
- [ ] Session starts are faster
- [ ] Context is automatically loaded

**Week 1 Success Criteria:**
- ✓ Memory system operational
- ✓ Session starts <1 minute
- ✓ At least 2 decisions recorded
- ✓ At least 2 patterns documented

## Week 2: Sprint Management

### Planning Day (2 hours)

- [ ] Copy sprint planner: `cp ~/mastering-claude-code/toolkit/sprint-planner.sh .`
- [ ] Make executable: `chmod +x sprint-planner.sh`
- [ ] Create docs directory: `mkdir -p docs`
- [ ] Create backlog: `./sprint-planner.sh backlog`
- [ ] Fill out BACKLOG.md with:
  - [ ] High priority stories (2-3 sprints worth)
  - [ ] Medium priority stories
  - [ ] Low priority stories
  - [ ] Story points estimated
  - [ ] Acceptance criteria defined

### Sprint 1 Planning (2 hours)

- [ ] Review last work to estimate velocity
- [ ] If no history, start conservative (15 points)
- [ ] Plan Sprint 1: `./sprint-planner.sh plan`
- [ ] Select stories totaling ~velocity points
- [ ] Break stories into tasks
- [ ] Define sprint goal
- [ ] Create `docs/SPRINT_01.md`

### Days 1-10: Sprint Execution (5 min/day)

- [ ] **Daily:** Update sprint progress: `./sprint-planner.sh update`
- [ ] **Daily:** Update task status with TaskUpdate()
- [ ] **Daily:** Track points completed
- [ ] **Day 5:** Mid-sprint checkpoint (adjust scope if needed)
- [ ] **Daily:** Commit after working states

### Day 10: Sprint End (2 hours)

- [ ] Sprint review: `./sprint-planner.sh review`
- [ ] Demo completed work
- [ ] Accept/reject stories
- [ ] Sprint retrospective: `./sprint-planner.sh retro`
- [ ] Record action items
- [ ] Update memory with learnings

### Week 2: Validation

- [ ] Sprint 1 completed
- [ ] SPRINT_01.md exists with full data
- [ ] RETROSPECTIVE_01.md exists
- [ ] Velocity measured
- [ ] Learnings captured in memory

**Week 2 Success Criteria:**
- ✓ First sprint completed
- ✓ Velocity measured (X points)
- ✓ Retrospective done
- ✓ Sprint 2 ready to plan

## Week 3: Task Integration

### Setup (30 minutes)

- [ ] Read `guides/TASK_INTEGRATION.md`
- [ ] Understand TaskCreate, TaskUpdate, TaskList, TaskGet
- [ ] Review task lifecycle (pending → in_progress → completed)

### Sprint 2 Planning with Tasks (2 hours)

- [ ] Plan Sprint 2: `./sprint-planner.sh plan`
- [ ] For each story, create tasks with TaskCreate():
  - [ ] Story 1 tasks created
  - [ ] Story 2 tasks created
  - [ ] Story 3 tasks created
- [ ] Set up task dependencies with TaskUpdate()
- [ ] Verify TaskList() shows all tasks

### Sprint 2 Execution (10 days)

- [ ] Use TaskUpdate() to set status=in_progress when starting
- [ ] Use TaskUpdate() to set status=completed when done
- [ ] Run TaskList() daily to see available tasks
- [ ] Let dependencies auto-unblock
- [ ] Track burndown via task completion

### Week 3: Validation

- [ ] TaskCreate() working
- [ ] TaskUpdate() working
- [ ] TaskList() shows correct state
- [ ] Dependencies functioning
- [ ] Burndown tracked automatically

**Week 3 Success Criteria:**
- ✓ Sprint 2 uses native tasks
- ✓ Dependencies set up correctly
- ✓ Task status tracked
- ✓ Burndown automated

- [ ] Update bead chain progress
- [ ] Track bead completion rate
- [ ] Measure actual vs estimated time per bead

### Week 4: Validation

- [ ] Beads executed successfully
- [ ] Commits after each bead
- [ ] Always working code
- [ ] Bead estimation improving

**Week 4 Success Criteria:**
- ✓ Sprint 3 uses bead method
- ✓ 8-12 commits per day
