# Part 1: The Basics

## 1. What Is "Vibe Coding"?

### The Old Way (Don't Do This)

**Before AI:**
1. Spend 6 months learning to code
2. Spend another 6 months learning frameworks
3. Spend another year getting good
4. Finally build something
5. Realize you wanted to build something else
6. Give up

**Success rate:** Like 5%. Maybe.

---

### The New Way (Vibe Coding)

**With AI:**
1. Describe what you want in plain English
2. Claude (your AI assistant) writes the code
3. You review it, test it, refine the vibe
4. It works
5. You ship it
6. You build the next thing

**Success rate:** Like 90% if you follow this guide.

---

### What "Vibe Coding" Actually Means

You know how interior designers just *vibe* with a space? They can't necessarily build the furniture, but they know exactly what the room should feel like, what mood it should have, what problem it solves.

**That's you with software now.**

You don't write code. You:
- **Describe the vibe** ("I want a dark mode dashboard that feels like a spaceship")
- **Test the feel** ("This button feels clunky, make it smoother")
- **Guide the direction** ("Users should feel confident, not overwhelmed")

Claude does the coding. You do the vibing.

## 3. Your AI Development Team

Meet your new employees. They work for free, never sleep, and never complain.

### The 10-Agent Dream Team

#### Tier 1: The Visionaries

**1. System Architect** (The Blueprinter)
- **Job:** Figures out how everything should fit together
- **When you need them:** Starting a new project
- **What they do:** "Okay, you want a bookmark manager. Here's how we'll structure it: a frontend in Next.js, a database in Supabase, authentication with Google sign-in..."
- **You interact how:** "I want this to work on mobile too" → They redesign the architecture

**2. Product Manager** (The Translator)
- **Job:** Breaks your dream into actual tasks
- **When you need them:** After architecture is done
- **What they do:** "To build this bookmark manager, we need: 1) Login page, 2) Main dashboard, 3) Add bookmark form, 4) Tag system, 5) Search function..."
- **You interact how:** "Which should we build first?" → They prioritize

#### Tier 2: The Designers

**3. Designer** (The Stylist)
- **Job:** Makes it look good and feel right
- **When you need them:** If you care about how it looks (and you should)
- **What they do:** "Dark mode with purple accents, clean minimalist cards for each bookmark, smooth animations when you save..."
- **You interact how:** "I want it to feel more playful" → They adjust the vibe

**4. API Architect** (The Data Plumber)
- **Job:** Designs how data flows through your app
- **When you need them:** Any app that saves or loads data
- **What they do:** "When user clicks Save, we'll POST to /api/bookmarks with the URL and tags, then GET all bookmarks on page load..."
- **You interact how:** You mostly don't — they work behind scenes

#### Tier 3: The Builders (Your Daily Crew)

**5. Frontend Developer** (The Face-Maker)
- **Job:** Builds everything you see and click
- **When you need them:** DAILY — This is your main agent
- **What they do:** Writes the actual buttons, forms, pages, animations
- **You interact how:** "Make this button bigger" / "Add a new page for settings" / "The spacing feels off"

**6. Backend Developer** (The Brain-Builder)
- **Job:** Builds the server, database, and logic
- **When you need them:** DAILY — Your other main agent
- **What they do:** Saves your bookmarks, handles logins, searches through data
- **You interact how:** "Users should be able to delete bookmarks" / "Add filtering by date"

#### Tier 4: The Quality Control

**7. Security Auditor** (The Protector)
- **Job:** Makes sure hackers can't mess with your app
- **When you need them:** Before launching anything public
- **What they do:** "Hey, users can edit OTHER people's bookmarks — that's a security hole. Fix it like this..."
- **You interact how:** You mostly don't — they just review and catch problems

**8. System Tester** (The Breaker)
- **Job:** Tries to break your app so users don't
- **When you need them:** After features are built
- **What they do:** "What if someone enters a 10,000 character URL? What if they're not logged in? Let me test everything..."
- **You interact how:** "Test the new tag system" → They write tests and report bugs

#### Tier 5: The Deployer

**9. DevOps Engineer** (The Publisher)
- **Job:** Puts your app on the internet
- **When you need them:** When you're ready to launch
- **What they do:** "Okay, we'll deploy to Vercel for the frontend, Railway for the backend, here's your live URL..."
- **You interact how:** "It's not working online but works on my computer" → They debug

#### Tier 6: The Documenter

**10. Technical Writer** (The Explainer)
- **Job:** Writes instructions for your app
- **When you need them:** If other people will use your app
- **What they do:** "How to add a bookmark: 1) Click the + button, 2) Paste your URL..."
- **You interact how:** "Write a README for this project" → They do it

### Step 2: Install The Basics

#### On Mac:

**2a. Open Terminal:**
- Press `Cmd + Space`
- Type "Terminal"
- Press Enter
- You'll see a black or white window with text

**2b. Install Homebrew** (Package Manager):
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

Copy that line, paste into Terminal (Cmd+V), press Enter.

**What this does:** Installs a "package manager" — think App Store but for developer tools.

**2c. Install Node.js** (JavaScript Runtime):
```bash
brew install node
```

**What this does:** Installs the engine that runs JavaScript apps.

**2d. Install Git** (Version Control):
```bash
brew install git
```

**What this does:** Installs your time machine / undo system.

**2e. Verify Everything Worked:**
```bash
node --version
git --version
```

You should see version numbers (like `v22.0.0` and `git version 2.40`).

**If you see version numbers:** ✅ You're done!

**If you see "command not found":** Close Terminal and open it again, then retry.

---

#### On Windows (if applicable):

**2a. Install Windows Subsystem for Linux (WSL):**

Open PowerShell as Administrator:
- Press `Windows Key`
- Type "PowerShell"
- Right-click "Windows PowerShell"
- Click "Run as Administrator"

Run this:
```bash
wsl --install
```

Restart your computer when it asks.

**What this does:** Gives you a proper Linux terminal inside Windows. Way better than Command Prompt.

**2b. After restart, open Ubuntu** (it auto-installed):
- Press `Windows Key`
- Type "Ubuntu"
- Open it
- Create a username and password when asked

**2c. Install Node.js:**
```bash
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

**2d. Install Git:**
```bash
sudo apt-get install git
```

**2e. Verify:**
```bash
node --version
git --version
```

You should see version numbers.

### The "Claude Manages These" Files

**What it is:** File where you put passwords, API keys, secrets

**What it is:** List of all code libraries your app needs

**You do:** Nothing. They just work in the background.

**Think of it like:** Airbags in a car

# Technical Specification — Bookmark Manager

## 12. When Things Break (They Will)

Let's talk about what to do when everything is on fire.

### Scenario 1: "It worked yesterday, now it's broken"

**What probably happened:** You changed something

**Fix:**
```bash
git log --oneline -10
```

This shows your last 10 commits (save points).

Find the last one where it worked:
```
abc1234 Added email feature
def5678 Fixed button styling  ← IT WORKED HERE
ghi9012 Tried to add dark mode toggle ← BROKE HERE
```

**Go back to that commit:**
```bash
git reset --hard def5678
```

**BOOM. It works again.**

**Now add dark mode properly** (ask Claude to help).

---

### Scenario 2: "Claude won't stop talking about something irrelevant"

**What happened:** Claude is confused or stuck in a loop

**Fix:**
- Close Claude Code (Ctrl+C)
- Update STATUS.md with what you're trying to do
- Restart Claude Code
- Be very specific: "Ignore previous conversation. Read STATUS.md. I need you to ONLY add a dark mode toggle to the settings page. Nothing else."

---

### Scenario 3: "I have NO idea what this error means"

**Example error:**
```
Error: Cannot find module '@/components/BookmarkCard'
```

**Ask Claude:**
```
I'm getting this error: [paste full error]

What does it mean and how do I fix it?
```

**Claude explains:**
```
This error means the app is looking for a file at `components/BookmarkCard.tsx` but can't find it. Either:

1. The file was deleted
2. The file is named wrong
3. The import path is wrong

Let me check... I see the file is actually named `BookmarkItem.tsx`. Let me fix the import.
```

**90% of errors = typos or missing files. Claude fixes them instantly.**

---

### Scenario 4: "Nothing works and I want to cry"

**The Nuclear Option:**

**Step 1: Breathe.**

**Step 2: Find your last working commit:**
```bash
git log --oneline -20
```

**Step 3: Go back to safety:**
```bash
git reset --hard <commit-where-it-worked>
```

**Step 4: Start fresh from there**

**Remember:** As long as you commit frequently, you can ALWAYS go back to a working state.

**That's the whole point of git.**

---

### The Golden Rule

**If you're stuck for more than 30 minutes:**

1. Check `TROUBLESHOOTING.md` in the framework
2. Ask Claude to explain the problem
3. Go back to last working commit
4. Take a break (seriously, walking helps)

**You're not dumb. Coding is weird. Things break randomly. Even senior developers Google errors daily.**

}

#### React Components

**What they are:** Reusable pieces of UI

**Example:**
```typescript
function Button({ label, onClick }) {

    <button onClick={onClick} className="bg-blue-500 text-white px-4 py-2 rounded">
      {label}
    </button>
  )

```

**You don't write these.** But you recognize them and can ask Claude to modify them.

---

#### API Routes

**What they are:** Server-side functions that handle data

**Example:**
```typescript
// app/api/bookmarks/route.ts

export async function GET() {
  const bookmarks = await database.getAll()
  return Response.json(bookmarks)
}

export async function POST(request) {
  const data = await request.json()
  await database.save(data)
  return Response.json({ success: true })
}
```

**What you need to know:**
- GET = fetch data
- POST = create data
- PUT/PATCH = update data
- DELETE = delete data

---

#### Database Queries

**What they are:** Commands that get/save data

**Example (Supabase):**
```typescript
// Get all bookmarks
const { data } = await supabase
  .from('bookmarks')
  .select('*')

// Get bookmarks with a specific tag
const { data } = await supabase
  .from('bookmarks')
  .select('*')
  .contains('tags', ['productivity'])

// Save new bookmark
await supabase
  .from('bookmarks')
  .insert({ url: '...', title: '...' })
```

**You don't write these.** But you can say: "Get only bookmarks with the 'work' tag" and Claude knows what to do.

**Fix:**

- "Done is better than perfect"
