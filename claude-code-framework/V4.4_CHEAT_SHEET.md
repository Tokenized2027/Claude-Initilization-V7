## Sprint Management

### Sprint Cycle
- **Day 0:** Planning (2 hours)
- **Days 1-10:** Execution (daily 5-min updates)
- **Day 10:** Review (1 hour) + Retrospective (1 hour)

### Commands
```bash
./essential/toolkit/sprint-planner.sh backlog  # Create backlog
./essential/toolkit/sprint-planner.sh plan     # Plan new sprint
./essential/toolkit/sprint-planner.sh update   # Daily progress
./essential/toolkit/sprint-planner.sh review   # Sprint review
./essential/toolkit/sprint-planner.sh retro    # Retrospective
./essential/toolkit/sprint-planner.sh status   # Current status
```

### Velocity Calculation
```
Velocity = avg points completed over last 3 sprints

Example:
  Sprint 1: 18 points
  Sprint 2: 22 points
  Sprint 3: 20 points
  Velocity: 20 points/sprint

Use 20 points for Sprint 4 capacity.
```

### Story Points
- **1 point:** 1-2 hours (simple)
- **2 points:** 2-4 hours (medium)
- **3 points:** 4-8 hours (complex)
- **5 points:** 8-16 hours (large)
- **8 points:** 16-24 hours (very large, consider splitting)
- **13+ points:** Too large, must split

## Task Integration

### Native Tools
```javascript
// Create task
TaskCreate({
  subject: "Brief imperative title",
  description: "Detailed what and why",
  activeForm: "Present continuous form (for spinner)",
  metadata: { sprint: "7", points: 3 }
})

// Update task
TaskUpdate({
  taskId: "1",
  status: "in_progress"  // or "completed"
})

// Set dependency (Task 2 depends on Task 1)
TaskUpdate({
  taskId: "2",
  addBlockedBy: ["1"]
})

// List all tasks
TaskList()

// Get task details
TaskGet({ taskId: "1" })
```

### Task Lifecycle
```
pending → in_progress → completed
         ↓
       deleted
```

### Task Discipline
- Set `in_progress` when starting
- Set `completed` when done
- Never leave tasks `in_progress` overnight

## Bead Method

### Bead Characteristics
- **Time:** 15-45 minutes
- **Output:** Working code
- **Commit:** One commit per bead
- **Value:** Incrementally valuable

### Decomposition Example
```
Feature: User Authentication (8 hours)

Beads:
1. User model (15 min) → commit
2. Password hashing (20 min) → commit
3. POST /register stub (20 min) → commit
4. Email validation (15 min) → commit
5. Password validation (15 min) → commit
6. POST /login stub (20 min) → commit
7. JWT generation (30 min) → commit
8. Token validation (25 min) → commit
9. Auth middleware (30 min) → commit
10. Integration test (30 min) → commit
11. Error handling (20 min) → commit

Total: 11 beads, ~4 hours, 11 commits, always working
```

### Bead Discipline
- Commit after each bead
- Keep beads 15-45 min
- Always working code
- If >45 min, split the bead

### Expected Commits
**8-12 commits/day** (6 hours / 30 min avg per bead)

## Integration Workflow

### Sprint Planning (Day 0)
1. Review backlog
2. Calculate velocity (last 3 sprints avg)
3. Select stories (~velocity points)
4. Break stories → tasks (TaskCreate)
5. Decompose tasks → beads (BEAD_CHAIN.md)
6. Commit sprint plan

### Daily Execution
```bash
# Morning
./essential/toolkit/memory-manager.sh start

# During day (repeat 8-12 times)
# 1. Execute one bead (15-45 min)
git add .
git commit -m "Brief description

- Change 1
- Change 2

Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>"

# 2. Update task if bead completes task
TaskUpdate({ taskId: "X", status: "completed" })

# Afternoon
./essential/toolkit/sprint-planner.sh update

# Evening
./essential/toolkit/memory-manager.sh end
```

### Sprint End (Day 10)
1. Review completed work
2. Accept/reject stories
3. Run retrospective
4. Record learnings → memory
5. Plan next sprint

## Troubleshooting

```bash
# Diagnostic
./essential/toolkit/validate-v4.4.sh

# Check memory files
ls -la .claude/memory/
jq . .claude/memory/decisions.json

# Check sprint status
cat docs/SPRINT_01.md

# Check git commits today
git log --since="today" --oneline

# Task status
TaskList()
```

### Common Issues
| Issue | Fix |
|-------|-----|
| Memory not loading | Re-paste load template in session |
| JSON parse error | Validate with `jq .` |
| Script not found | `chmod +x essential/toolkit/*.sh` |
| Tasks not persisting | Tasks are session-specific, recreate |
| Beads too long | Split into smaller beads |
| Velocity unstable | Lock sprint scope, consistent estimation |
