**Before v4.4:**

## The Four Pillars

### 1. Memory System

**What it does:**
- Captures decisions with rationale
- Learns patterns (bugs, solutions)
- Remembers your preferences
- Tracks session history
- Enables cross-project learning

**Files:**
```
.claude/memory/
├── decisions.json       # Architectural decisions
├── patterns.md          # Learned patterns
├── preferences.md       # Your coding style
├── project-context.json # Current state snapshot
└── session-history.json # Last 10 sessions
```

**Commands:**
```bash
memory-manager.sh start    # Load memory, start session
memory-manager.sh end      # Save memory, end session
memory-manager.sh sync     # Sync to global memory
```

**ROI:** 10x time invested

---

### 2. Sprint Management

**What it does:**
- 2-week sprint cycles
- Velocity-based planning
- Daily progress tracking
- Sprint review & demo
- Retrospectives with action items

**Files:**
```
docs/
├── BACKLOG.md           # Product backlog
├── SPRINT_01.md         # Sprint plan
└── RETROSPECTIVE_01.md  # Sprint retro
```

**Commands:**
```bash
sprint-planner.sh plan     # Plan new sprint
sprint-planner.sh update   # Daily progress
sprint-planner.sh review   # Sprint review
sprint-planner.sh retro    # Retrospective
```

**ROI:** Predictable delivery, continuous improvement

---

### 3. Task Integration

**What it does:**
- Uses Claude Code's built-in task tools
- Structured task creation
- Dependency management
- Multi-agent coordination
- Automated burndown

**Tools:**
```javascript
TaskCreate()  // Create trackable tasks
TaskUpdate()  // Update status, dependencies
TaskList()    // View all tasks
TaskGet()     // Get task details
```

**Integration:**
- Tasks map to sprint stories
- Dependencies block/unblock automatically
- Burndown calculated from task completion

**ROI:** Better visibility, no missed dependencies

---

### 4. Bead Method

**What it does:**
- Breaks features into 15-45 min increments
- Each bead = working code + commit
- Continuous integration
- Visible progress
- Reduced risk

**Template:**
```markdown
Bead 1: User model (15 min) → commit
Bead 2: Password hashing (20 min) → commit
Bead 3: Register endpoint (25 min) → commit
...
Total: 5 hours / 12 beads
```

**Benefits:**
- Always working code
- Low sunk cost (easy pivots)
- Granular progress tracking
- Reduced stress

## Technical Details

### Memory System Architecture

```
User preferences → preferences.md (manual config)
    ↓
Decisions → decisions.json (auto-capture via hooks)
    ↓
Patterns → patterns.md (manual + auto-extracted)
    ↓
Project state → project-context.json (auto-updated)
    ↓
Session history → session-history.json (auto-logged)
    ↓
Global memory → ~/.claude/global-memory/ (cross-project)
```

### Sprint System Architecture

```
Product backlog → BACKLOG.md
    ↓
Sprint planning → SPRINT_XX.md (with TaskCreate)
    ↓
Daily execution → memory-manager.sh start/end
    ↓
Progress tracking → sprint-planner.sh update
    ↓
Sprint review → Acceptance/rejection
    ↓
Retrospective → RETROSPECTIVE_XX.md
    ↓
Memory update → Patterns, velocity, action items
    ↓
Next sprint planning (with learnings)
```
