### 1.3 Firewall (UFW + Docker Fix)

CRITICAL: Docker bypasses UFW by default by manipulating iptables directly. This means even with UFW rules set to deny incoming, any Docker container port exposed with `-p` or `ports:` in docker-compose is accessible from your entire LAN. We must fix this.

- Configure UFW:
  - Default deny incoming, allow outgoing
  - Allow SSH (port 22)
  - Allow Tailscale (port 41641/udp)
  - Do NOT open HTTP/HTTPS (80, 443) — all services are Tailscale-only [DECISION CONFIRMED: Tailscale-only]
- Fix Docker/UFW conflict using ONE of these approaches (recommend Option A):
  - **Option A (recommended):** In ALL docker-compose files, bind exposed ports to 127.0.0.1 only:
    ```yaml
    ports:
      - "127.0.0.1:8080:8080"  # Only accessible from localhost and Tailscale
    ```
  - **Option B:** Set `"iptables": false` in /etc/docker/daemon.json (can break container-to-container networking — more complex)
- Enable UFW
- Verification: `sudo ufw status verbose`, confirm only SSH and Tailscale are open. From another device on the same LAN (not Tailscale), confirm you CANNOT access any Docker service ports.
